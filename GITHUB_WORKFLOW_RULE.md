# GitHub Workflow Rule

This file sets the working rule for how MTC files should be managed between live working folders and GitHub.

## Core Rule

- `iCloud / local working folders` = live daily work
- `GitHub` = stable systems, templates, reusable structure, and versioned project assets

## Use iCloud Or Local Working Folders For

- weekly lesson preparation
- daily teaching documents
- active class files
- student-specific notes
- working drafts
- live admin material
- anything that changes frequently during the week

## Use GitHub For

- classroom systems
- onboarding processes
- role descriptions
- reusable worksheets
- branded templates
- reusable PowerPoint assets
- process documents
- stable prompts
- safe scripts
- project guidance files

## Simple Decision Rule

Ask:

`Am I doing today's live work, or am I building a reusable system?`

- If it is today's live work, keep it in `iCloud / local working folders`
- If it is a reusable system, keep it in `GitHub`

## Promotion Rule

Use this sequence:

1. Create and work on the item in the live working environment first.
2. When it becomes stable, reusable, or worth version history, move or copy it into GitHub.
3. Treat the GitHub version as the maintained systems copy.

## Avoid

- editing the same file in both places
- using GitHub as a dumping ground
- storing sensitive student evidence in GitHub
- putting fast-changing daily admin work into GitHub
- guessing which environment to use

## Monthly Review Habit

Once each month:

1. Review what has been created recently.
2. Keep live operational work in the working environment.
3. Promote reusable items into GitHub.
4. Clean up duplicates.

## MTC Repo Intent

This MTC repository is the `systems layer` for:

- `04_Classroom_Activities/`
- `07_Templates/`
- `templates/presentations/`
- selected safe files in `generated/`
- `AGENTS.md`

It is not the main storage place for student evidence, daily admin, or sensitive operational material.
